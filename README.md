# Trivia Risk: Margin of Error

A Risk-style world-conquest game where **dice combat is replaced by numeric trivia duels**.
Every territory has a fixed trivia category; when you attack, the defending territory's
category sets the question. A brand-new question is generated by the **Anthropic API** for
every battle, and the game remembers past questions so they never repeat. Troops still
matter — each committed troop shaves 10 "error points" off your guess — but a sharp
estimate can still beat a bigger army. **Margin of error decides the war.**

2–4 **human** players, local hot-seat. No AI.

## How a battle works

1. Attacker picks an adjacent enemy territory and how many troops to commit (1–4).
2. Defender's troops are committed (default `min(2, troops)`; selectable).
3. The server asks Anthropic for a fresh question in the defending territory's category.
4. Attacker enters a numeric guess; then the defender (who sees the attacker's guess) enters theirs.
5. The answer is revealed and scored:
   - **Amount** questions: `errorPoints = (max/min − 1) × 100` (scale-independent).
   - **Year** questions: `errorPoints = |guess − answer| × 5`.
   - **Troop bonus:** `(committedTroops − 1) × 10`, subtracted from error points (floored at 0).
   - Lowest final score wins; exact ties go to the **defender**.
6. Damage by score gap: ≤20 → 1 troop, ≤60 → 2, >60 → 3 (capped at the loser's committed troops).
   Reducing a defending territory to 0 troops captures it.

## Run it

```bash
npm install
cp .env.example .env        # then add your ANTHROPIC_API_KEY
npm start                   # serves http://localhost:4318
```

Open **http://localhost:4318**, set up 2–4 players (names + colors), and play.

### `ANTHROPIC_API_KEY`
Set it in `.env` (server-side only — it is **never** sent to the browser). The default
question model is `claude-opus-4-8`; override with `ANTHROPIC_MODEL` (e.g. `claude-haiku-4-5`).
There is a developer-only `ALLOW_FALLBACK_QUESTIONS=true` flag that returns clearly-marked
TEST-ONLY questions when generation fails — leave it **off** for real play.

## Question uniqueness

Every battle requests a *new* question (no preset bank, no caching). Before generating, the
server passes recent history to Anthropic — the **last 20 questions in the same category** and
the **last 50 overall** — instructing it not to repeat the topic, entity, fact, answer, or
wording. After Anthropic responds, the server runs a local duplicate check
([`shared/questions.js`](shared/questions.js): exact text, fingerprint of
`category|entity|factType|answer`, same entity+fact, same entity+answer, keyword overlap).
On a duplicate it asks Anthropic to regenerate, up to 3 times. If generation fails, the
battle shows **"Question generation failed. Try again."** with a *Retry question* button and a
*Cancel attack* option — the battle never advances on a bad question, and the game never crashes.

## Save / load

The server is authoritative and **auto-saves after every action** to `server/data/current.json`
(reloading the browser resumes mid-game). Use **Save** to write a named slot and **Load** to
restore one (`server/data/saves/<name>.json`). A saved game includes players, current player,
turn, phase, ownership, troop counts, territory categories, any pending battle, the full used-
question history + fingerprints, and the battle/event history.

## Architecture

Authoritative game state and rules live on the server; the browser is a thin renderer that
POSTs actions and re-renders the returned state. The Anthropic key and unrevealed answers are
never exposed to the client.

| Path | Purpose |
|------|---------|
| [`shared/map.js`](shared/map.js) | 42-territory map, adjacency, positions, the 8 categories |
| [`shared/scoring.js`](shared/scoring.js) | Margin-of-error math (pure, tested) |
| [`shared/questions.js`](shared/questions.js) | Fingerprint + duplicate/near-duplicate detection |
| [`shared/rules.js`](shared/rules.js) | Authoritative game engine (setup, reinforce, attack, fortify, turns, win) |
| [`server/anthropic.js`](server/anthropic.js) | Server-side question generation + uniqueness retry |
| [`server/store.js`](server/store.js) | File-based persistence (autosave + named saves) |
| [`server/index.js`](server/index.js) | Express API + static hosting |
| [`public/`](public/) | Client: SVG map, battle modal, history, save/load |
| [`test/scoring.test.js`](test/scoring.test.js) | Scoring + duplicate-detection tests (`npm test`) |

## The 8 trivia categories

World & Geography · History & Dates · Science & Nature · Sports ·
Entertainment & Pop Culture · Business & Money · Food & Everyday Life · Wild Stats

Categories are assigned **randomly** to territories at game start (not by real-world region)
and stay fixed for the saved game.
